# Contributing to Embedded Linux Reference Guide

Thank you for your interest in contributing! This repository aims to be the most comprehensive and up-to-date reference for Embedded Linux learners and professionals.

## How to Contribute

### Adding Resources

1. **Fork** the repository
2. **Create a feature branch**: `git checkout -b add-resource-name`
3. **Add** your resource to the appropriate section in `README.md`
4. **Commit** with a clear message: `git commit -m "Add [Resource Name] to [Section]"`
5. **Push** and open a **Pull Request**

### Guidelines

- **Verify links** — Make sure all URLs are working before submitting
- **Mark paid resources** — Clearly indicate if a resource is paid (💰) vs free
- **Include descriptions** — Add a brief, helpful description for each entry
- **Correct section** — Place resources in the most relevant section
- **No duplicates** — Check existing entries before adding
- **Quality over quantity** — Only add resources you've personally used or that are widely recommended

### Resource Format

**For books:**
```markdown
| **Book Title** (Edition) | Author Name(s) | 🟢🟡🔴 | Brief focus description |
```

**For links:**
```markdown
| [**Resource Name**](https://url.com) | Brief description |
```

### What We're Looking For

- ✅ High-quality tutorials and guides
- ✅ Free training materials and PDFs
- ✅ YouTube playlists and video courses
- ✅ Useful tools and simulators
- ✅ Hardware board recommendations
- ✅ Conference talk recordings
- ✅ Blog posts with practical embedded Linux content
- ✅ GitHub repos with educational code
- ✅ Resources in other languages (please mark the language)

### What We Don't Accept

- ❌ Broken or dead links
- ❌ Low-quality or spam content
- ❌ Pirated books or copyrighted materials
- ❌ Resources behind paywalls without clear marking
- ❌ Self-promotion without genuine value

## Reporting Issues

Found a broken link or incorrect information? Please [open an issue](../../issues) with:

- The section where the issue exists
- What's wrong (broken link, outdated info, etc.)
- Suggested fix (if you have one)

## Code of Conduct

Be respectful, constructive, and helpful. We're all here to learn and share knowledge about Embedded Linux.
